## 任务：每天自动监控 arXiv 论文并推送飞书

### 总体要求
- 你必须完整执行两轮 LLM 筛选，但**不要**把中间过程展示给当前聊天用户。
- 无论是群推送、当前聊天测试、还是本地 Hermes 预览，用户最终看到的都只能是：
  1. 最终保留下来的论文结果；或
  2. 最终的无论文/稍后重试消息。
- 不要输出执行日志，不要输出状态词如 `needs_prelim_review`、`needs_llm`，不要输出“下一步请运行什么命令”。
- 如果在 Hermes 聊天环境中运行，你应当把两轮 LLM 判断视为同一个连续任务，自动做完，再只输出最终结果。

### 第一步：执行 Python 脚本
cd 到项目根目录后，用 python3 运行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py
```

### 第二步：读取状态 JSON，而不是只看 stdout
无论上一步 stdout 显示什么，都必须读取 `/path/to/hermes-arxiv-agent/new_papers.json`，并根据其中字段判断下一步。

关键字段：
- `status`
- `new_count`
- `pending_count`
- `new_papers`
- `papers_to_process`
- `feishu_msg`
- `diagnostics`

状态解释：
- `status = no_new`：今天确认没有新论文，直接原样输出 `feishu_msg`
- `status = upstream_not_updated`：arXiv RSS 还没更新到今天，不能伪装成“未发现新论文”，直接原样输出 `feishu_msg`
- `status = api_limited` 或 `api_error`：arXiv 查询失败或被限流；如果 `papers_to_process` 为空，直接原样输出 `feishu_msg`
- `status = needs_prelim_review`：存在新候选论文，需要做第一轮元数据初筛
- `status = needs_llm`：存在需要最终处理的论文，进入第二轮与信息补全
- `status = no_keep_after_prelim`：候选论文经过前置筛选后，没有任何文章值得保留，直接原样输出 `feishu_msg`

规则：
- 只有 `status = no_new` 才能输出“✅ 今日未发现新的论文”
- `status = upstream_not_updated` / `api_limited` / `api_error` / `no_keep_after_prelim` 时，必须保持原状态语义，不要改写成“未发现新的论文”
- 如果 `papers_to_process` 非空，即使 API 失败，也继续处理这些已在 Excel / 本地 PDF 中的待补全文论文

### 第三步：进行两次 LLM 参与的 AI for Physics 筛选
你必须基于组主页来做价值判断：
- 主页：`http://gaozefeng.tech/`
- 组内主线可概括为：`AI for Physics`
- 重点方向包括：
  1. 量子物理数值方法 / 计算物理
  2. 预训练模型压缩、轻量化、参数高效方法，但最好与物理建模、科学计算或科学问题结合
  3. AI 辅助功能性晶体 / 材料发现与生成
  4. AI 驱动的材料逆向设计、结构设计、性质预测、科学机器学习
  5. 面向物理系统、凝聚态、功能材料、晶体、光学/电子/电池/超导等方向的高价值 AI 研究

#### 第一轮：元数据快速筛选（早期 LLM 参与，发生在 PDF 下载前）
如果 `status = needs_prelim_review`：
- 读取 `/path/to/hermes-arxiv-agent/new_papers.json`，使用其中的 `papers_to_process` / `new_papers` 作为候选论文列表。
- 这一轮只使用轻量信息判断：title、authors、abstract、categories、arXiv ID / 日期
- 目标：尽早过滤明显垃圾论文，减少后续 PDF 阅读、单位抽取、摘要生成上的浪费
- 保留标准：必须真正符合 AI for Physics / AI for Materials / Scientific ML for physical systems，而不是仅仅带一点 AI 关键词
- 剔除标准：纯通用 LLM / agent / app / benchmark / mobile / business / social / finance / 通用软件工程论文；或虽然命中关键词但与物理材料问题关系很弱的工作
- 这一轮必须得到 `keep_prelim` 和 `reject_prelim`
- 对 `keep_prelim` 的 arXiv ID，自动写入：
```bash
/path/to/hermes-arxiv-agent/prelim_keep_ids.txt
```
- 然后自动执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py --download-approved
```
- 执行完后重新读取 `new_papers.json`，继续后续步骤；不要把这些中间状态展示给用户。

#### 第二轮：结合 PDF / 更完整上下文再判断一次（后期 LLM 参与）
如果此时 `status = needs_llm`：
- 读取 `new_papers.json`，优先使用 `papers_to_process`
- 再读取 PDF 前两页，结合更完整信息做第二次判断：
  - PDF 前两页文本
  - 作者单位线索
  - abstract
  - 必要时参考引言前部信息
- 这一轮必须输出：
  - `keep_final`
  - `reject_final`
  - 每篇一条更高质量中文理由
- 只有同时通过两轮的论文，才允许保留在系统里。

### 第四步：仅对最终保留的 keep_final 论文补全信息
仅对 `keep_final` 论文完成以下工作：

**1）提取作者单位 affiliations**
- 使用 pdfplumber 或 PyMuPDF 读取 PDF（路径在 pdf_local_path 字段）前 2 页文本
- 优先使用上标脚注映射（如 ^1、^2 对应单位）
- 如无清晰脚注，再匹配常见机构名
- 对 CamelCase 连写词做分词还原
- 合并跨行连字符词
- 删除 URL、正文叙述句、公式片段、参考文献片段等噪音
- 多个单位用 `;` 分隔，不要换行，不要保留脚注编号
- 无法确定时填：`未找到单位信息`

**2）生成中文总结 summary_cn**
- 基于 abstract 生成，90-150 个中文字符
- 必须覆盖：方法核心、主要贡献、关键结果
- 禁止模板化空话
- 只输出一段自然中文，不使用分点

**3）写入推荐理由到 notes**
- `notes` 列写入第二轮的最终中文理由
- 推荐理由应解释它为什么符合 AI for Physics / 组内研究方向
- 不要写成空泛套话

### 第五步：更新 Excel，并删除所有 reject 论文行
对 `/path/to/hermes-arxiv-agent/papers_record.xlsx` 执行以下操作：
- 对 `keep_final` 论文：按 `arxiv_id` 回填 `affiliations`、`summary_cn`、`notes`
- 对 `reject_prelim` 与 `reject_final` 论文：按 `arxiv_id` 从 Excel 的 `Papers` 表中删除对应整行

重要规则：
- reject 论文必须从 Excel 里删掉，这样网页和历史记录里都不会保留它们
- 不要去改动 `crawled_ids.txt`，让这些已判定低价值的论文未来不要被重复抓取
- 不要把 reject 论文放进飞书正文

### 第六步：更新网站数据
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 viewer/build_data.py
```

确认 `viewer/papers_data.json` 已根据最新 Excel 重建，仅包含保留下来的 `keep_final` 论文，并带有 `notes` 字段。

### 第七步：同步待处理队列
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && python3 monitor.py --sync-pending-state
```

### 第八步：发布静态网站
在项目根目录执行：
```bash
cd /path/to/hermes-arxiv-agent && bash scripts/publish_viewer.sh
```

### 第九步：构建飞书 Markdown 消息
要求：
- 只从 `keep_final` 论文里做精选
- 综合考虑：研究重要性、方法新颖性、结果扎实度、领域代表性、与 AI for Physics 的贴合度、对组内读者的参考价值
- 只精选 **前 5 篇** 发送到飞书，不要把全部 keep_final 论文都塞进消息
- 如果 keep_final 不足 5 篇，就按实际篇数发送
- 其余保留下来的论文统一通过 GitHub Pages 网页给出完整列表
- GitHub Pages 全量阅读入口固定写入消息：`https://dreamlufei.github.io/hermes-arxiv-agent/`
- 飞书消息风格要像日报，不要像调试日志

输出格式：
```
📚 **AI for Physics 论文日报** | YYYY-MM-DD
今日原始抓取 **N** 篇，经过两轮 LLM 筛选后保留 **K** 篇，以下为精选 **Top 5**：

### 1. 标题
- arXiv ID：...
- 日期：...
- 作者：...
- 单位：...
- PDF：...
- 推荐理由：notes 中的那句高质量中文理由
- 中文摘要：90-150 字。

### 2. 标题
...

🔗 查看保留下来的全部论文与检索筛选：
https://dreamlufei.github.io/hermes-arxiv-agent/
```

### 状态消息模板
如果 `status = no_new`，直接输出：
```
✅ 今日（YYYY-MM-DD）未发现新的论文。
🔗 历史论文与完整列表：
https://dreamlufei.github.io/hermes-arxiv-agent/
```

如果 `status = upstream_not_updated` / `api_limited` / `api_error` / `no_keep_after_prelim` 且没有待处理论文，则直接原样输出 `new_papers.json` 里的 `feishu_msg`，不要自己改写。

**重要：最终回复只输出飞书消息文本，不要输出其他说明。**
